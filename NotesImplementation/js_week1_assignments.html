P-01 Electricity Bill Calculator
Easy
Variables Data types if/else Operators
Problem

A power company charges electricity based on slabs. Write a program that takes units consumed and prints the full bill breakdown.

Slab rates
  • 0 – 100 units → ₹3 per unit
  • 101 – 200 units → ₹5 per unit (for units above 100)
  • 201 – 300 units → ₹7 per unit (for units above 200)
  • Above 300 → ₹10 per unit (for units above 300)
  • Add 18% GST on the final amount
Expected output (for 250 units)

Base charge: ₹1150
GST (18%): ₹207
Total bill: ₹1357
Hint: Use separate if-else blocks for each slab. Compute the charge slab by slab and add them up — don't multiply all units by the highest slab rate.

Problem

A store has an inventory stored as a nested object. Write four separate functions to manage it.

Inventory data
const inventory = {
  apple:   { price: 20,  stock: 150, category: "fruit"    },
  banana:  { price: 10,  stock: 8,   category: "fruit"    },

P-09 Number Analyzer
Hard
Operators Conditionals while loop for loop Functions Arrow functions
Problem

Write an analyzeNumber(n) function that fully analyzes a given positive integer. Write each check as a separate named arrow function, then call them all inside analyzeNumber.

Checks to implement
  • isPrime(n) — a number with no divisors other than 1 and itself. 1 is not prime.
  • isPerfect(n) — sum of all divisors (excluding itself) equals the number. Example: 6 → divisors are 1,2,3 → sum=6 ✓
  • isArmstrong(n) — sum of each digit raised to the power of total digits equals n. Example: 153 → 1³+5³+3³ = 153 ✓
  • isPalindrome(n) — number reads same forwards and backwards. Example: 121 ✓
  • getDivisors(n) — returns an array of all divisors of n
Expected output (for 153)
Analyzing: 153
Is Prime:      false
Is Perfect:    false
Is Armstrong:  true
Is Palindrome: false
Divisors:      [1, 3, 9, 17, 51, 153]
Hint: For isPrime, loop from 2 to √n (use Math.sqrt()). For isArmstrong, convert n to a string to find its length (number of digits), then extract digits using the % and Math.floor() operators in a while loop. For isPerfect, loop from 1 to n-1 and collect divisors.
P-10 Cricket Scorecard Generator
Hard
Variables Conditionals for...of Nested loops Functions Rest params
Problem

You are given a ball-by-ball commentary array. Each entry is a string representing what happened on that ball. Generate a complete scorecard from it.

Ball outcomes (strings)
  • "0" – dot ball (no runs)
  • "1", "2", "3", "4", "6" – runs scored
  • "W" – wicket (no runs, adds 1 wicket)
  • "WD" – wide (1 extra run, ball not counted in over)
  • "NB" – no ball (1 extra run, ball not counted in over)
Match data
const balls = [
  "1","4","0","W","2","6",
  "WD","3","0","4","1","W","6",
  "0","0","NB","4","6","0","1",
  "W","4","1","0","6","2",
];
What to compute and print
  • Total runs scored
  • Total wickets fallen
  • Total extras (wides + no balls)
  • Total legal balls bowled and overs completed (e.g. "4.2 overs")
  • Run rate (total runs / overs as decimal)
  • Over-by-over summary: which runs were scored in each over
  • Highest scoring over
  • Total fours and sixes hit
Expected output format
==== SCORECARD ====
Total:        87/3
Overs:        4.0
Run Rate:     21.75
Extras:       2 (2 WD, 0 NB)
Fours: 5  Sixes: 3

-- Over Summary --
Over 1: 13 runs  [1,4,0,W,2,6]
Over 2: ...
Best over: Over 1 (13 runs)
Hint: Maintain a legalBalls counter (only increments on non-wide, non-no-ball deliveries). Every 6 legal balls = 1 over. Use a nested array to group balls by over. Use continue inside your loop to skip extras from the ball count.
